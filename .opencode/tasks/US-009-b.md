# US-009-b · View + Client-side (template + Alpine.js + CSS)

## Objetivo

Implementar a view de edição do perfil: template Thymeleaf `perfil/adolescente-editar.html` com avatar block, campos nickname/birthDate/email(readonly), save bar sticky com dirty state detection via Alpine.js, modal de bloqueio por idade. CSS para avatar-block, save-bar, modal. i18n messages em pt-BR.

Depende de: US-009-a (controller precisa existir para renderizar e POST).

## Contexto

### Arquivos relevantes
| Arquivo | Ação |
|---|---|
| `src/main/resources/templates/perfil/adolescente-editar.html` | **novo** |
| `src/main/resources/static/css/app.css` | editar — adicionar avatar-block, save-bar, modal styles |
| `src/main/resources/messages.properties` | editar — adicionar chaves de UI e validação do perfil |

### Padrão escolhido
Template Thymeleaf decorando `layout/base.html` (Layout Dialect). Alpine.js para dirty state detection (`x-data`, `@input`, `x-show`). CSS puro seguindo tokens de design system existentes. O spec UX (`doc/UX/us-009-spec.md`) define microcopy literal em pt-BR — copiar exatamente como está.

### Alternativas descartadas
1. **HTMX para dirty state** — rejeitado: HTMX é ideal para swap de fragments, mas dirty detection requer comparação client-side de valores originais vs atuais — Alpine.js com `x-data` é mais natural e já está incluído no base.html.
2. **Tailwind classes novas** — rejeitado: o projeto usa CSS custom com tokens; adicionar Tailwind utilities seria inconsistente com o padrão existente em `app.css`.

## Testes

> Esta subtask é view/frontend. No padrão atual do projeto, templates Thymeleaf + CSS não têm testes unitários dedicados (testes são MockMvc para controllers). A renderização é validada visualmente pelo revisor e pela integração com o controller (US-009-a já tem MockMvc tests que cobrem GET/POST `/perfil`).

1. **Arquivo** `src/main/resources/templates/perfil/adolescente-editar.html` · **Validação:** template deve compilar sem erros Thymeleaf quando acessado via `GET /perfil` (controller US-009-a já existe). Modelo esperado: `{form: UpdateProfileForm, email: String, avatarUrl: String}`.
2. **Arquivo** `src/main/resources/static/css/app.css` · **Validação:** CSS deve ser válido (sem erros de sintaxe); classes `.avatar-block`, `.save-bar`, `.age-modal-overlay` devem existir e seguir tokens do design system.

### 1. Criar (novo) `src/main/resources/templates/perfil/adolescente-editar.html`

**Conteúdo inteiro:**

```html
<!DOCTYPE html>
<html lang="pt-BR" xmlns:th="http://www.thymeleaf.org"
      xmlns:layout="http://www.ultraq.net.nz/thymeleaf/layout"
      layout:decorate="~{layout/base}">

<th:block layout:fragment="content">

<!-- Header público com botão voltar -->
<header th:replace="~{layout/fragments/public-header :: public-back-header('/trilha', 'Voltar à trilha')}"></header>

<main class="app-main profile-edit-page">

  <!-- Banner de sucesso (flash message) -->
  <div class="success-banner" x-data="{ show: false }"
       x-show="show" x-init="if (window.location.search.includes('saved')) { show = true; setTimeout(() => show = false, 4000); }">
    <p>Alterações salvas! Teu progresso continua igual.</p>
  </div>

  <!-- Avatar block -->
  <div class="avatar-block" x-data="{ actionSheetOpen: false }">
    <div class="avatar-circle" th:classappend="${avatarUrl != null && !avatarUrl.isBlank()} ? 'has-photo' : ''">
      <img th:if="${avatarUrl != null && !avatarUrl.isBlank()}"
           th:src="${avatarUrl}"
           alt="Foto de perfil"
           class="avatar-img"/>
      <span th:unless="${avatarUrl != null && !avatarUrl.isBlank()}"
            th:text="${#strings.substring(form.nickname, 0, 1)}"
            class="avatar-initial"></span>
      <!-- Badge de câmera (abre action sheet — US-009-c) -->
      <button class="camera-badge"
              @click="actionSheetOpen = true"
              aria-label="Alterar foto">
        <svg viewBox="0 0 24 24" fill="none" stroke="currentColor"
             stroke-width="2" stroke-linecap="round" stroke-linejoin="round"
             width="16" height="16" aria-hidden="true">
          <path d="M23 19a2 2 0 0 1-2 2H3a2 2 0 0 1-2-2V8a2 2 0 0 1 2-2h4l2-3h6l2 3h4a2 2 0 0 1 2 2z"/>
          <circle cx="12" cy="13" r="4"/>
        </svg>
      </button>
    </div>
    <h1 class="profile-nickname" th:text="${form.nickname}"></h1>
    <p class="profile-email">
      <span th:text="${email}"></span>
      <svg viewBox="0 0 24 24" fill="none" stroke="currentColor"
           stroke-width="2" width="14" height="14" aria-hidden="true" class="lock-icon">
        <rect x="3" y="11" width="18" height="11" rx="2" ry="2"/>
        <path d="M7 11V7a5 5 0 0 1 10 0v4"/>
      </svg>
    </p>
  </div>

  <!-- Form de edição -->
  <form th:action="@{/perfil}" method="post" th:object="${form}"
        x-data="{
          originalNickname: '${form.nickname}',
          originalBirthDate: '${form.birthDate}',
          dirty: false,
          ageModalOpen: false,

          markDirty() {
            this.dirty = (this.$refs.nickname.value !== this.originalNickname) ||
                         (this.$refs.birthDate.value !== this.originalBirthDate);
          },

          discard() {
            this.$refs.nickname.value = this.originalNickname;
            this.$refs.birthDate.value = this.originalBirthDate;
            this.dirty = false;
          }
        }"
        @submit.window="if ($event.target === $el) { markDirty(); }">

    <!-- Nickname -->
    <div class="form-group">
      <label for="nickname" class="form-label">Apelido</label>
      <input type="text" id="nickname" name="nickname"
             th:field="*{nickname}"
             x-ref="nickname"
             @input="markDirty()"
             minlength="3" maxlength="20"
             class="form-input"
             :class="{ 'has-error': $refs.nickname.errors?.length > 0 }"/>
      <p class="form-helper">Entre 3 e 20 caracteres.</p>
      <p class="form-error" th:if="${#fields.hasErrors('nickname')}" th:errors="*{nickname}"></p>
    </div>

    <!-- BirthDate -->
    <div class="form-group">
      <label for="birthDate" class="form-label">Data de nascimento</label>
      <input type="date" id="birthDate" name="birthDate"
             th:field="*{birthDate}"
             x-ref="birthDate"
             @input="markDirty()"
             class="form-input"
             :class="{ 'has-error': $refs.birthDate.errors?.length > 0 }"/>
      <p class="form-helper">Só é possível editar se a idade continuar entre 13 e 17 anos.</p>
      <p class="form-error" th:if="${#fields.hasErrors('birthDate')}" th:errors="*{birthDate}"></p>
    </div>

    <!-- Email (readonly) -->
    <div class="form-group">
      <label for="email" class="form-label">E-mail</label>
      <input type="email" id="email" name="email"
             th:value="${email}"
             readonly
             class="form-input form-input--readonly"/>
      <p class="form-helper">É o que você usa pra entrar na atrilha. Mudar exige nova verificação.</p>
    </div>

  </form>

</main>

<!-- Save bar sticky (aparece só quando dirty) -->
<div class="save-bar" x-data="{ visible: false }"
     x-show="visible"
     x-init="$watch('dirty', val => visible = val)"
     :class="{ 'save-bar--visible': visible }">

  <div class="save-bar__inner">
    <button type="button" class="btn btn--ghost" @click="$refs.form.discard()">
      Descartar
    </button>
    <button type="submit" form="profile-form" class="btn btn--primary">
      Salvar alterações
    </button>
  </div>
</div>

<!-- Modal de bloqueio por idade -->
<div class="age-modal-overlay"
     x-data="{ open: false }"
     x-show="open"
     x-transition:enter="transition ease-out duration-200"
     x-transition:enter-start="opacity-0"
     x-transition:enter-end="opacity-100"
     x-transition:leave="transition ease-in duration-150"
     x-transition:leave-start="opacity-100"
     x-transition:leave-end="opacity-0"
     @keydown.escape.window="open = false; $refs.form.discard()"
     style="display: none;">

  <div class="age-modal" role="alertdialog" aria-labelledby="age-modal-title">
    <h2 id="age-modal-title">Ops, essa data não funciona</h2>
    <p id="age-modal-message">
      A atrilha é para adolescentes de 13 a 17 anos. Se você tentar salvar com uma data fora dessa faixa, a gente não deixa — é pra te proteger!
    </p>
    <button type="button" class="btn btn--primary btn--full"
            @click="open = false; $refs.form.discard()">
      Entendi, manter como estava
    </button>
  </div>

</div>

<!-- Action sheet (US-009-c — placeholder estrutural) -->
<div class="action-sheet-overlay"
     x-data="{ open: false }"
     x-show="open"
     style="display: none;">
  <!-- Conteúdo será adicionado na US-009-c -->
</div>

</th:block>

</html>
```
